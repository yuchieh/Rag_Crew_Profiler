# 環境建置與本地模型對接指南 (Milestone 1)

歡迎來到這學期 **AgentSociety Challenge** 的第一步。

在開始建構能讀取與分析 Yelp 評論的 Multi-Agent 系統前，我們需要先將開發環境與底層模型設定妥當。在實作過程中，大家可能會遇到套件版本衝突、驗證報錯，或是免費 API 請求次數受限 (Rate Limit) 等問題。

為了解決這些常見的初期障礙，我整理了這份**「本機端、免費、無限制」的起步方案**。只要依照以下步驟進行設定，就能順利避開環境配置上經常出現的錯誤，讓大家能專注於優化 Prompt 與系統架構。

---

## 步驟一：使用 uv 進行環境建置

在這門課裡，我建議使用 `uv` 來管理套件，以取代傳統的 `pip install` 與 `requirements.txt`。`uv` 是目前業界廣泛使用的高效套件管理工具。

我已經為大家準備好了對齊過底層版本的 `pyproject.toml` 與 `uv.lock`（各位可以在 `docs/hw_supports/` 資料夾下找到，請將這兩個檔案複製到專案的根目錄）。

**🛠️ 執行步驟：**
1. 確認電腦中已安裝 `uv` (Mac 用戶可使用指令 `brew install uv`)。
2. 開啟終端機，在專案根目錄下輸入以下指令：
   ```bash
   uv sync
   ```
> [!TIP]
> 該指令會自動建立乾淨的虛擬環境，並下載我們已測試穩定的 `crewai`, `langchain-community`, `sentence-transformers` 等相依套件，確保大家的執行環境保持一致。

---

## 步驟二：安裝本地語言模型 Ollama (解決 429 請求限制)

在開發推論型 Agent 時，系統會在短時間內頻繁與語言模型傳遞資訊。如果使用免費版的雲端 API（例如 Gemini Flash 或 Groq），很容易在一分鐘內耗盡免費請求額度，導致程式出現 `429 Rate Limit Error` 或 `RESOURCE_EXHAUSTED` 的錯誤而中斷。

**為了解決這個問題，我們將改用完全在本機執行的語言模型。**

**🛠️ 執行步驟：**
1. 前往 [Ollama 官方網站](https://ollama.com/download) 下載並安裝應用程式。
2. 啟動 Ollama 後，開啟終端機，輸入下方指令以下載微軟的 `Phi-3` 模型：
   ```bash
   ollama run phi3
   ```
*(等待下載完成，顯示 `>>>` 提示字元後即可按下 `Ctrl+D` 退出。此模型未來會在背景伺服器中隨時待命。)*

---

## 步驟三：設定 `.env` 環境變數

CrewAI 的底層套件需要正確的環境變數才能順利運行。請在專案根目錄建立（或修改）`.env` 檔案，輸入以下內容：

```env
# 指定所有 Agent 使用剛下載好的 Ollama 本機模型
MODEL=ollama/phi3

# 【重要】繞過 CrewAI-Tools 底層早期強制檢查 OpenAI Key 的問題
# 加上這行能避免 Agent 讀取 Knowledge 時，錯誤地向 OpenAI 發送請求而產生 401 Unauthorized 報錯。
OPENAI_API_KEY=NA

# 如果有註冊免費的 Serper 帳號用於網路搜尋工具，可以在這裡放置 金鑰 (選填)
SERPER_API_KEY=你的SerperKey
```

## 步驟四：RAG 檢索工具的正確設定與綁定

本系統需要從 Yelp 的 JSON 資料集取得情報，因此我們會使用 `JSONSearchTool`。以下提出兩點常見且容易造成系統崩潰的設定錯誤，請務必參照下方的範例程式碼進行撰寫：

1. **嚴格的套件參數驗證**：在最新版的工具庫中，直接傳入 LangChain 的物件會導致驗證失敗，請改用 Dictionary 指定純本機的 Sentence-Transformer 來建立推論切塊。
2. **工具命名衝突**：在向不同的 Agent 掛載 RAG 工具時，若工具的名稱 (`.name`) 相同，Function Calling 機制會無法分辨，進而引發 `400 INVALID_ARGUMENT` 錯誤。

**🛠️ `crew.py` 設定範例：**

以下是將上述避坑原則整合後的 `crew.py` 頂層實作參考，請確保您的程式碼包含這些關鍵的初始化步驟：

```python
import os
from langchain_community.embeddings import HuggingFaceEmbeddings
from crewai_tools import JSONSearchTool

# 1. 繞過 CrewAI-Tools 底層強制檢查 OpenAI Key 的問題
os.environ["OPENAI_API_KEY"] = "NA"

# 2. 設定全域的 Embedding 模型 (用於 CrewAI Knowledge 等底層檢索)
embedding_model = HuggingFaceEmbeddings(
    model_name='BAAI/bge-small-en-v1.5'
)

# 3. 專門提供給 RAG Tool 使用的設定檔 (字典格式)
# 由於我們在 .env 設定了 MODEL=ollama/phi3，此處無需特別指定 LLM Provider，工具會自動回退使用 Ollama。
rag_config = {
    "embedding_model": {
        "provider": "sentence-transformer", # 指定使用純本機端 CPU 自動建立推論切塊
        "config": {
            "model_name": "BAAI/bge-small-en-v1.5"
        }
    }
}

# 4. 【重要】確保為每一個檢索工具設定獨立的名稱 (.name) 與描述 (.description)
user_rag_tool = JSONSearchTool(json_path='data/user_subset.json', collection_name='v3_hf_user_data', config=rag_config)
user_rag_tool.name = "search_user_profile_data"
user_rag_tool.description = "Useful to retrieve a specific user's giving habits, average stars, and review counts."

item_rag_tool = JSONSearchTool(json_path='data/item_subset.json', collection_name='v3_hf_item_data', config=rag_config)
item_rag_tool.name = "search_restaurant_feature_data"
item_rag_tool.description = "Useful to retrieve a specific restaurant's location, categories, attributes, and overall stars."

review_rag_tool = JSONSearchTool(json_path='data/review_subset.json', collection_name='v3_hf_review_data', config=rag_config)
review_rag_tool.name = "search_historical_reviews_data"
review_rag_tool.description = "Useful to retrieve the actual text content of past reviews for users or restaurants."
```

👉 **`uv run first_crew`**

執行初期，程式會花費幾分鐘為 Yelp 資料集進行編碼並建立本地向量庫。當此階段完成後，即可在畫面上觀察到 3 位 AI Agent 的合作與決策過程，並最終為你產出高品質的推薦預測報表。
